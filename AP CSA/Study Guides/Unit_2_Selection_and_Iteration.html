Topic 2.1

Algorithms with Selection and Repetition

含选择与重复的算法

Algorithms are built from three fundamental blocks:

算法（algorithm）由三个基本结构构成：

Sequencing, steps execute in order, one at a time.

Selection, a choice is made based on a true/false decision.

Repetition, a process repeats until a desired outcome is reached.

顺序：各步骤依次执行，每次执行一条。

选择：根据 true/false 判断做出分支。

重复：某段过程不断重复，直到达成目标。

Key Insight The order in which sequencing, selection, and repetition are combined determines the algorithm's outcome.
关键点 顺序、选择、重复这三种结构组合的次序决定了算法的最终结果。

Topic 2.4

Nested if Statements

嵌套 if（nested if）语句

An if inside another if. The inner condition is only checked if the outer condition is true.

一个 if 嵌套在另一个 if 中。只有外层条件为 true 时，才会检查内层条件。

if-else-if (Multiway Selection)

if-else-if（多分支选择）

if (grade >= 90) {
    letter = "A";
} else if (grade >= 80) {
    letter = "B";
} else if (grade >= 70) {
    letter = "C";
} else {
    letter = "F";
}
Multiway Selection Rule Only the first branch whose condition is true executes. All remaining branches are skipped. If none are true and there's a trailing else, that else runs.
多分支选择规则 只有第一个条件为 true 的分支会执行，后面的分支全部跳过。若所有条件都不成立且末尾有 else，则执行该 else。
Topic 2.5

Compound Boolean Expressions

复合布尔表达式

Logical Operators

逻辑运算符（logical operator）

OperatorNameTrue When
!NOT!a is true when a is false
&&ANDa && b is true only when both are true
||ORa || b is true when at least one is true
运算符名称何时为真
!非（NOT）当 a 为假时，!a 为真
&&与（AND）当两者都为真时，a && b 才为真
||或（OR）当至少一个为真时，a || b 为真

Precedence: ! → && → ||

优先级（precedence）： ! → && → ||

int age = 20;
boolean hasID = true;
if (age >= 18 && hasID) {
    System.out.println("Entry allowed");
}
Short-Circuit Evaluation Java stops evaluating as soon as the result is determined: with &&, if the first operand is false, the second is never checked. With ||, if the first operand is true, the second is skipped. This is important when the second expression could cause an error (e.g., null check).
短路求值（short-circuit evaluation） Java 一旦能确定整体结果就停止继续求值： 对 &&，若左操作数为 false，右操作数不再检查； 对 ||，若左操作数为 true，右操作数被跳过。 当右侧表达式可能抛错时（例如 null 判空），这点尤其重要。
Worked Example: Short-circuit guards against NullPointerException
String s = null;

// Wrong order: s.length() runs first, throws NullPointerException
if (s.length() > 0 && s != null) { ... }

// Correct order: null check first, short-circuit skips s.length()
if (s != null && s.length() > 0) { ... }

Why this matters on the exam: AP MCQs love to put the null check second, so the dangerous method call happens first. Short-circuit evaluation only saves you if the guard is on the left side of &&. Same idea with ||: put the "safe" condition that can short-circuit out the dangerous one on the left.

例题：用短路求值防止 NullPointerException
String s = null;

// 顺序错误：先执行 s.length()，抛出 NullPointerException
if (s.length() > 0 && s != null) { ... }

// 顺序正确：先判空，短路会跳过 s.length()
if (s != null && s.length() > 0) { ... }

为什么这在考试上重要： AP 选择题最爱把 null 判断放在第二位，让危险的方法调用先发生。只有把守卫条件放在 && 的左侧，短路求值才能救你。|| 同理：把可以短路掉危险条件的"安全"判断放在左边。

AP Trap: Operator precedence inside if-conditions ! binds tightest, then &&, then ||. So a || b && c is read as a || (b && c), not (a || b) && c. When in doubt, add parentheses. The AP rubric does not subtract points for "redundant" parentheses, but it does subtract for wrong groupings.
AP 陷阱：if 条件中的运算符优先级 ! 结合最紧，其次是 &&，最后是 ||。所以 a || b && c 应解读为 a || (b && c)，而不是 (a || b) && c。拿不准时就加括号。AP 评分标准不会因"多余"的括号扣分，但会因错误分组扣分。

Comparing Boolean Expressions

比较布尔表达式

De Morgan's Law

De Morgan's Law, Must Memorize
OriginalEquivalent
!(a && b)!a || !b
!(a || b)!a && !b

Flip the operator (&& ↔ ||) and negate each operand.

德摩根定律，必背
原式等价式（equivalent）
!(a && b)!a || !b
!(a || b)!a && !b

翻转运算符（&& ↔ ||），并对每个操作数取反。

Comparing Objects

比较对象

ComparisonWhat It Checks

obj == null引用是否未指向任何对象？

String b = new String("Hi");

a.equals(b)  // true , same content// true，内容相同
Which is equivalent to !(x > 5 && y < 3)?
下列哪个与 !(x > 5 && y < 3) 等价？
x > 5 || y < 3
!x > 5 && !y < 3
x <= 5 || y >= 3
x <= 5 && y >= 3
Correct! By De Morgan's: negate each condition and flip && to ||. !(x>5) → x<=5, !(y<3) → y>=3, and && → ||.
正确！由德摩根定律：对每个条件取反，并把 && 翻成 ||。!(x>5) → x<=5，!(y<3) → y>=3，&& → ||。
Apply De Morgan's: negate each part and flip the operator. !(x>5 && y<3) = x<=5 || y>=3.
运用德摩根定律：每一项取反，运算符翻转。!(x>5 && y<3) = x<=5 || y>=3。
Worked Example: Negating a compound condition

You want the opposite of "the student passed AND submitted on time".

// Original
passed && onTime

// Negated (apply De Morgan: flip && to ||, negate each side)
!passed || !onTime

Read aloud: "the student failed OR was late". That matches your intuition for the opposite of "passed AND on time". If it doesn't match intuition, you probably flipped the operator without negating the operands (or vice versa).

例题：对复合条件取反

你想得到"学生通过 AND 按时提交"的反面。

// 原式
passed && onTime

// 取反（用德摩根：&& 翻成 ||，每一项取反）
!passed || !onTime

读出来：「学生不及格 OR 没按时」。这正是"通过 AND 按时"的反面，与直觉一致。如果不一致，说明你只翻转了运算符却没把操作数取反（或者反过来）。

AP Trap: Negating relational operators The negation of x > 5 is x <= 5, not x < 5. Always include the boundary value in the negation. Same logic: !(x == 5) becomes x != 5; !(x >= 5) becomes x < 5. Drop the "or equal" half exactly once per flip.
AP 陷阱：关系运算符取反 x > 5 的反是 x <= 5，不是 x < 5。取反时必须把边界值包进来。同理：!(x == 5) 变为 x != 5；!(x >= 5) 变为 x < 5。每次翻转时，"等于"那一半要恰好被丢掉一次。
Topic 2.7

while Loops

while 循环（while loop）

Iteration repeats code zero or more times while a condition is true.

循环（loop / iteration）会在条件为真时重复执行代码零次或多次。

int count = 1;
while (count <= 5) {
    System.out.print(count + " ");
    count++;
}
// Output: 1 2 3 4 5// 输出：1 2 3 4 5
while Loop Mechanics The Boolean expression is checked before each iteration (including the first). If it's initially false, the body never executes.
while 循环的执行机制 每次迭代之前（包括第一次）都会检查布尔表达式。如果一开始就为 false，循环体一次都不会执行。
Infinite Loop If the condition never becomes false, the loop runs forever. Always ensure the loop body changes something that will eventually make the condition false.
死循环（infinite loop） 如果条件永远不会变成 false，循环就会一直跑下去。务必确保循环体中有某个变化最终能让条件为 false。
Off-by-One Errors Using < vs <= (or starting at 0 vs 1) can cause a loop to run one time too many or too few. Trace carefully!
差一错误（Off-by-One） 用 < 还是 <=（或从 0 起还是从 1 起）会让循环多跑一次或少跑一次。一定要仔细追踪！
Worked Example: Tracing a while loop
int n = 20;
int count = 0;
while (n > 1) {
    n /= 2;
    count++;
}
// What is count?

Final answer: count = 4. This is essentially counting how many times you can halve n until it drops to 1, which is roughly log2(n). A common AP pattern.

AP Trap: Reading the variable's value at the wrong moment When tracing a loop, distinguish "value at the start of iteration" from "value after the body". The condition is checked at the start of each pass, using the value from the end of the previous pass. Forgetting this is the most common source of off-by-one mistakes in MCQ traces.
AP 陷阱：在错误的时间点读变量值 追踪循环时，要区分"迭代开始时的值"与"循环体执行完后的值"。条件是在每一轮的开头检查的，使用的是上一轮末尾的那个值。忽视这一点是选择题追踪中差一错误的最常见原因。
Topic 2.8

for Loops

for 循环（for loop）

A for loop packs initialization, condition, and update into one header.

一个 for 循环把初始化、条件和更新打包在同一个头部里。

for (int i = 0; i < 5; i++) {
    System.out.print(i + " ");
}
// Output: 0 1 2 3 4// 输出：0 1 2 3 4

Three Parts of a for Header:

for (initialization; Boolean expression; update)

1. Init runs once before the loop starts.
2. Condition is checked before each iteration.
3. Update runs after each iteration's body, before the next check.

for 头部的三个部分：

for (初始化; 布尔表达式; 更新)

1. 初始化：循环开始前只执行一次。
2. 条件：每次迭代前都会检查。
3. 更新：每次循环体之后、下一次检查之前执行。

for ↔ while Equivalence Every for loop can be rewritten as a while loop and vice versa:
for 与 while 的等价性 每个 for 循环都能改写成 while 循环，反之亦然： 
// Equivalent while loop:// 等价的 while 循环：
int i = 0;
while (i < 5) {
    System.out.print(i + " ");
    i++;
}
How many times does this loop iterate?
for (int i = 3; i < 8; i++)
这个循环会迭代多少次？
for (int i = 3; i < 8; i++)
5 times (i = 3, 4, 5, 6, 7)5 次（i = 3, 4, 5, 6, 7）
6 times (i = 3, 4, 5, 6, 7, 8)6 次（i = 3, 4, 5, 6, 7, 8）
4 times (i = 3, 4, 5, 6)4 次（i = 3, 4, 5, 6）
Infinite loop死循环
Correct! i starts at 3 and runs while i < 8, so it takes values 3, 4, 5, 6, 7 → 5 iterations.
正确！i 从 3 起，当 i < 8 时执行，因此取值 3, 4, 5, 6, 7 → 5 次迭代。
Count: 3, 4, 5, 6, 7 (stops before 8 because condition is < not <=). That's 5 iterations.
数一遍：3, 4, 5, 6, 7（因为条件是 < 而不是 <=，所以到 8 之前就停了）。共 5 次。
Worked Example: Final value of a loop variable
for (int i = 0; i < 5; i++) {
    // body runs with i = 0, 1, 2, 3, 4
}
// after the loop, i = 5  (but only if i was declared OUTSIDE the for header)

The update step runs after every iteration, including the one that fails the condition. So i reaches the first value that makes the condition false. With i < 5, that's i = 5. With i <= 5, it's i = 6.

Catch: when i is declared inside the for header (for (int i = 0; ...)), it goes out of scope when the loop ends, and you can't read its final value. Declare i outside the header if you need it afterward.

例题：循环变量（loop variable）的最终值
for (int i = 0; i < 5; i++) {
    // 循环体在 i = 0, 1, 2, 3, 4 下执行
}
// 循环结束后，i = 5（仅当 i 在 for 头部之外声明时可见）

更新步骤在每次迭代后都会执行，包括让条件变假的那一次。所以 i 会到达让条件首次为假的那个值。i < 5 时是 i = 5；i <= 5 时是 i = 6。

注意： 当 i 在 for 头部内声明（for (int i = 0; ...)）时，循环结束后 i 就出了作用域，你拿不到它的最终值。如果之后还要读它，就把 i 在头部外声明。

AP Trap: Iteration count formulas For for (int i = a; i < b; i++) with a < b: iteration count is b - a. For i <= b: count is b - a + 1. For i += step: count is ceil((b - a) / step). Memorising the closed-form is faster than tracing the full loop on a timed MCQ.
AP 陷阱：迭代次数公式 对于 for (int i = a; i < b; i++)（a < b）：次数为 b - a。对 i <= b：次数为 b - a + 1。对 i += step：次数为 ceil((b - a) / step)。在限时选择题里，背公式比完整追踪更快。

Topic 2.11

Nested Iteration

嵌套循环（nested loop）

A loop inside another loop. The inner loop completes all its iterations for each single iteration of the outer loop.

循环里再套循环。外层每跑一次，内层都要把它的所有迭代跑完。

for (int row = 1; row <= 3; row++) {
    for (int col = 1; col <= 4; col++) {
        System.out.print("* ");
    }
    System.out.println();
}
// Output:
// * * * *
// * * * *
// * * * *// 输出：
// * * * *
// * * * *
// * * * *
Counting Iterations If the outer loop runs m times and the inner loop runs n times, the inner body executes m × n times total. Above: 3 × 4 = 12 stars.
计算迭代次数 如果外层跑 m 次、内层跑 n 次，那么内层体总共执行 m × n 次。上例：3 × 4 = 12 个星号。
Worked Example: When the inner loop depends on the outer index
for (int i = 1; i <= 4; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("*");
    }
    System.out.println();
}

Output:

*
**
***
****

When j's upper bound depends on i, you can't use the m × n shortcut. Total stars: 1 + 2 + 3 + 4 = 10, not 4 × 4 = 16. The shortcut only works when the inner loop bounds are constant across outer iterations.

例题：内层循环依赖外层索引
for (int i = 1; i <= 4; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("*");
    }
    System.out.println();
}

输出：

*
**
***
****

当 j 的上界依赖于 i 时，不能套用 m × n 的捷径。星号总数：1 + 2 + 3 + 4 = 10，不是 4 × 4 = 16。这个捷径只在内层边界对每次外层迭代都不变时才适用。

AP Trap: Print vs println in nested loops The newline behaviour is the most common nested-loop MCQ trap. println after the inner loop produces a row-and-column grid; println inside the inner loop puts each star on its own line; missing the println entirely produces one long row. Re-read where the newlines actually fall before picking an answer.
AP 陷阱：嵌套循环里的 print 与 println 换行的位置是嵌套循环选择题最常见的陷阱。println 放在内层循环之后会形成行列网格；放在内层循环之内会让每个星号自占一行；完全没有 println 则全部挤成一长串。选答案前先把换行实际出现的位置看清楚。
Topic 2.12

Informal Run-Time Analysis

非形式化运行时分析

A statement execution count is the number of times a statement runs. You determine this by tracing the loop.

语句执行次数指一条语句被执行的次数。通过追踪循环来确定它。

AP MCQ Preview

Unit 2 MCQ Patterns That Show Up Every Year

单元 2 每年都考的选择题套路

Unit 2 is the most-tested unit on the AP CSA MCQ section. Most questions fall into a handful of recurring shapes. Recognising the shape first lets you trace less and answer faster.

单元 2 是 AP CSA 选择题中考得最多的单元。大多数题都落在几种固定的套路里。先认出套路，就能少追踪、快出答案。

Pattern 1: Iteration counting (closed form vs trace) The question gives a for-loop header and asks how many times the body runs. For i < b starting at a, the count is b - a. For i <= b, it's b - a + 1. For i += step, count is ceil((b - a) / step). Skip the trace if you can spot the form. (See Topic 2.8.)
套路 1：迭代（iteration）计数（公式法 vs 追踪法） 题目给出 for 循环头部，问循环体执行多少次。从 a 起的 i < b：次数为 b - a。i <= b：次数为 b - a + 1。i += step：次数为 ceil((b - a) / step)。能套公式就不要全程追踪。（见 2.8 节）
Pattern 2: Short-circuit and null safety Compound conditions with a null guard. The correct AP answer always has the null check on the left of && (or the "safe" condition that short-circuits). Order-of-operands MCQs are common. (See Topic 2.5.)
套路 2：短路求值（short-circuit evaluation）与 null 安全 带 null 判空的复合条件。正确的 AP 答案永远把 null 判断放在 && 的左侧（或者把能短路掉危险条件的"安全"判断放在左侧）。操作数顺序型选择题很常见。（见 2.5 节）
Pattern 3: De Morgan's transforms "Which expression is equivalent to !(...)?" The mechanical answer: flip && ↔ || and negate each operand. Watch the relational flips: !(x > 5) is x <= 5, not x < 5. (See Topic 2.6.)
套路 3：德摩根变换（De Morgan's Laws） "下列哪个与 !(...) 等价？" 机械做法：把 && ↔ || 翻过来，并对每个操作数取反。注意关系运算符的翻转：!(x > 5) 是 x <= 5，不是 x < 5。（见 2.6 节）
Pattern 4: Nested loop with dependent inner bound Inner loop runs j ≤ i or j < n - i. Total iterations form a triangular sum (1 + 2 + ... + n = n(n+1)/2) or a reversed triangle. The m × n shortcut does not apply. (See Topic 2.11.)
套路 4：内层边界依赖外层的嵌套循环（nested loop） 内层循环跑 j ≤ i 或 j < n - i。总迭代次数构成三角和（1 + 2 + ... + n = n(n+1)/2）或反向三角。这种情况下 m × n 的捷径不适用。（见 2.11 节）
Pattern 5: Off-by-one boundary picking Two answer choices differ only by < vs <=. Trace the boundary cases: what happens at the first and last value? The "correct" formula usually pairs with the inclusive or exclusive endpoint of the problem statement. (See Topic 2.7.)
套路 5：差一边界的选项辨别 两个选项只在 < 与 <= 上有差别。追踪边界情况：第一个值和最后一个值各会怎样？"正确"公式通常与题面里端点是闭还是开相匹配。（见 2.7 节）
Pattern 6: Trace-the-output for nested print/println Where the newlines fall determines whether you see a triangle, a square, or a single long line. Mark every println with a forced line break and trace iteration by iteration. (See Topic 2.11.)
套路 6：嵌套 print/println 的输出追踪 换行落在哪里决定了你看到的是三角形、正方形还是一长行。把每个 println 都标记为强制换行，然后一轮一轮地追踪。（见 2.11 节）
